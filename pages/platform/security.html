Security Features

Enterprise-grade controls and capabilities.

1

Identity & Access Management

SSO (SAML/OIDC), SCIM provisioning, MFA, short-lived tokens. For agents: mTLS with X.509 certificates. RBAC & ABAC for fine-grained authorization.

2

Network Segmentation

Agent / Orchestrator / Governor / Execution / Management segments. Agents restricted to Orchestrator only. Explicit firewall rules between layers.

3

Encryption & Key Management

TLS 1.3 for transit, AES-256-GCM at rest. KMS/HSM integration. BYOK support for sovereign deployments. Automated key rotation.

4

Policy Package Signing

Policies are cryptographically signed. Governor validates signature before execution. Tampered packages are rejected.

5

Audit & Compliance

Append-only audit records with policy version hash, user (pseudonymized), action, decision. Replayable for forensics and auditor review.

6

Observability & Monitoring

Structured JSON logs, Prometheus metrics, distributed tracing. Integration with SIEM and APM stacks. Alerting on security events.

Security Deployment Checklist

Quick reference for security architects and infrastructure teams.

Pre-Deployment Validation

  • Network segmentation per reference architecture (agent ‚Üí orchestrator ‚Üí governor ‚Üí execution)
  • TLS 1.3 enforced across all components; mTLS for service identities
  • SSO + SCIM for user provisioning; MFA for admin access
  • KMS/HSM integration and (optional) BYOK configured
  • Centralized logging with immutable audit storage
  • Policy CI/CD (tests + manual compliance gate + signing)
  • Signed packages for air-gapped/on-prem updates
  • Incident response plan and security contact established

Security FAQ

Answers to common security and compliance questions.

Can we use our own KMS / BYOK?

Yes ‚Äî BYOK is supported for sovereign and on-prem deployments. Key operations remain under customer control.

What happens if Governor is unreachable?

Default is fail-closed (block actions). Fail-open is configurable per workflow with explicit audit logging.

Do audit logs contain PII?

Audit logs use pseudonyms by default. PII is stored separately with policies supporting deletion/pseudonymization.

Can we run Axiosky air-gapped?

Yes ‚Äî signed packages and offline verification workflows support air-gapped deployments.

How do you prove auditability?

Each decision includes policy version/hash and rationale. The system can re-execute policy code to reproduce decisions.

Are penetration test reports public?

Executive summaries are shared under NDA to qualified customers. Regular testing is conducted by third parties.
